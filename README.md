# CALENDAR
Тестирование приложения CALENDAR_1.1.
## Корректный ввод
#### Тест 1: Ввод диапазона от 01.01.0001 до 31.12.5000
Ожидаемый результат: Программа выводит дату время из указанного диапазона.
#### Тест 2: Ввод диапазона от 01.01.5001 до 31.12.9999
Ожидаемый результат: Программа выводит дату время из указанного диапазона.
## Некорректный ввод
#### Тест 3: Ввод диапазона от 31.12.5000 до 01.01.0001
Ожидаемый результат: Программа выдает сообщение об ошибке "Введенные значения неверные. Попробуйте еще раз".
#### Тест 4: Ввод диапазона с невалидными датой (например от 01.01.0001 до 30.02.2040)
Ожидаемый результат: Программа выдает сообщение об ошибке "Введенные значения неверные. Попробуйте еще раз".
#### Тест 5: Ввод диапазона с невалидными датой (например от 01.13.0001 до 01.02.2040)
Ожидаемый результат: Программа выдает сообщение об ошибке "Введенные значения неверные. Попробуйте еще раз".
#### Тест 6: Ввод диапазона с использованием других символов вместо точки (например от 01,01,0100 до 01,02,1040)
Ожидаемый результат: Программа выдает сообщение об ошибке "Введенные значения неверные. Попробуйте еще раз".
#### Тест 7: Ввод диапазона в другом формате  (например от 2000.01.27 до 2001.07.10)
Ожидаемый результат: Программа выдает сообщение об ошибке "Введенные значения неверные. Попробуйте еще раз".
## Описание найденных багов
#### Баг 1: Некорректный вывод даты при правильном диапазоне дат.
###### Описание:
При вводе корректных значений дат, программа выдает результат в формате ГГГГ-ММ-ЧЧ.
###### Шаги для воспроизведения:
Ввести 1 дату: 01.01.2000. 
Нажать кнопку "Enter". ![image](https://github.com/user-attachments/assets/4fc975ed-1b0d-4a65-902d-cb0e28c7f61c)
Ввести 2 дату: 31.12.2000.
Нажать кнопку "Enter". ![image](https://github.com/user-attachments/assets/2635bc1f-270f-4aa6-b4b2-541216810ecf)
###### Ожидаемый результат:
Вывод рандомной даты и времени в написанном диапозоне в формате ЧЧ.ММ.ГГГГ.
###### Фактический результат:
Программа выдает рандомную дату и время в написанном диапозоне в формате ГГГГ-ММ-ЧЧ. ![image](https://github.com/user-attachments/assets/8ae28e4b-aba7-476e-96d1-1760fe2d13f7)
#### Баг 2: Ошибка вывода времени при правильном диапазоне дат.
###### Описание: 
Программа выдает время в несуществующем диапозоне, показывает более 23:59:59.999.
###### Шаги для воспроизведения:
Ввести 1 дату: 01.01.2000.
Нажать кнопку "Enter". ![image](https://github.com/user-attachments/assets/1c03c5f4-8a86-4774-b726-41094ed25300)
Ввести 2 дату: 31.12.2000.
Нажать кнопку "Enter". ![image](https://github.com/user-attachments/assets/77af137e-2a41-403e-a754-0fd6cbdf6405)
Для воспроизведения бага иногда придется повторить несколько раз. ![image](https://github.com/user-attachments/assets/8ae28e4b-aba7-476e-96d1-1760fe2d13f7)
###### Ожидаемый результат:
Вывод рандомного времени в написанном диапозоне в формате ЧЧ:ММ:СС.МСК. ≤ 23:59:59.999
###### Фактический результат:
Программа может выдать рандомное время в формате ЧЧ:ММ:СС.МСК > 23:59:59.999
## Заключение
